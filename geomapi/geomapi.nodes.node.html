geomapi.nodes.node

Node is an abstract Python Class to govern the data and metadata of remote sensing data (pcd, images, meshes, orthomozaik). It is the base class for all other node classes. It contains the base RDF graph functionality and i/o from and to RDF files.

IMPORTANT: the Node class is an archytype class from which specific data classes (e.g. PointCloudNode) inherit. Do not use this class directly if you can use a child class with more functionality.

Goals:

  • Govern the metadata and geospatial information of big data files in an accesible and lightweight format.

  • Serialize the metadata of remote sensing and geospatial files (BIM, Point clouds, meshes, etc.) as RDF Graphs

  • Attach spatial en temporal analyses through RDF Graph navigation

Creates a Node from one or more of the following inputs.

Args:

1.graph (Graph, optional): An RDF Graph to parse.

2.graphPath (str, optional): The path of an RDF Graph to parse. If no subject is provided, the first subject of the graph is retained.

3.path(str,optional): A filepath to a resource.

4.subject (URIRef, optional): A subject to use as identifier for the Node. If a graph is also present, the subject should be part of the graph.

5.name (str, optional): A name of the Node. This is not a unique identifier but serves as non-functional description.

property cartesianTransform

Get the Cartesian Transform (translation & rotation matrix) (np.ndarray(4x4)) of the node. Note that the initialisation from different inputs may result in different cartesianTransform values.

E.g. the pose of a mesh is retrieved from the mean of the vertices,

while the same pose initiated from the cartesianBounds equals the mean of that cartesianBounds array.

If no cartesianTransform is present, you can use get_cartesianTransform() to construct the cartesiantransform from the resource, cartesianBounds, orientedBounds or orientedBoundingBox.

Features:

  1. self.cartesianBounds

  2. self.resource

  4. self.orientedBoundingBox
